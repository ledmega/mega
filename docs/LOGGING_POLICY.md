# MEGA 시스템 로그 운영 정책

이 문서는 **MEGA WebServer**와 **Agent**의 로그 수집, 보관 및 관리 정책을 설명합니다.

---

## 1. 개요 (Common Features)
*   **로그 저장 경로**: 각 실행 디렉토리 하위의 `./logs/` 폴더
*   **로테이션 주기**: **일 단위 (Daily)** (매일 00:00에 파일 분리)
*   **보관 기간**: **최대 30일** (30일이 지난 로그는 자동 삭제)
*   **출력 타임존**: **Asia/Seoul (KST)**
*   **인코딩**: **UTF-8**

---

## 2. MEGA WebServer 로그 (메인 서버)

WebServer는 업무 성격에 따라 로그 파일을 분리하여 관리함으로써 장애 대응 및 감사 추적의 효율성을 높이고 있습니다.

| 로그 파일명 | 용도 및 수집 대상 | 상세 설명 |
| :--- | :--- | :--- |
| **`mega.log`** | 전체 운영 로그 (Root) | 시스템 전반의 INFO 급 이상 일반 정보 |
| **`mega-error.log`** | 시스템 에러 로그 | **ERROR 레벨**만 별도 추출 (긴급 대응용) |
| **`mega-batch.log`** | 배치 작업 로그 | 스케줄링된 배치 작업의 시작/종료 및 결과 |
| **`mega-agent.log`** | 에이전트 통신 로그 | 에이전트와의 API 통신 및 SSE 실시간 알림 이력 |
| **`mega-security.log`** | 보안 및 인증 로그 | 로그인/로그아웃, 권한 위반 등 보안 관련 이벤트 |
| **`mega-cs-ai.log`** | **CS & AI 엔진 로그** | AI 응답 생성 과정, 톡드림 연동, 시뮬레이션 데이터 |

### [용량 제한 정책]
*   **전체 로그(mega.log)**: 최대 10GB 제한
*   **개별 로그(Error, CS, Security 등)**: 각 파일당 최대 5GB 제한

---

## 3. Agent 로그 (수집 에이전트)

에이전트는 로컬 장비의 데이터를 수집하고 전송하는 역할을 수행하므로, 네트워크 상태와 태스크 실행 수집 이력에 집중합니다.

| 로그 파일명 | 용도 및 수집 대상 | 상세 설명 |
| :--- | :--- | :--- |
| **`agent.log`** | 전체 대기 로그 | 에이전트 실행 상태 요약 |
| **`agent-error.log`** | 실행 에러 로그 | 수집 실패 및 런타임 익셉션 이력 |
| **`agent-data.log`** | 데이터 수집/파싱 로그 | 수집 대상 데이터의 파싱 및 전송 전처리 상세 이력 |
| **`agent-net.log`** | 네트워크 상태 로그 | 서버와의 연결 상태(Heartbeat), 전송 성공/실패 여부 |
| **`agent-task.log`** | 스케줄링 태스크 로그 | 개별 수집 태스크(Executor)의 실행 및 스케줄링 이력 |

### [용량 제한 정책]
*   **기본 로그(agent.log)**: 최대 1GB 제한 (장비 리소스 보존을 위해 서버보다 낮게 설정)

---

## 4. 로그 레벨 운영 가이드

1.  **INFO (기본)**: 시스템 흐름 파악을 위한 주요 이벤트 기록
2.  **DEBUG**: 개발 및 로컬 테스트 시 상세 로직 추적용 (설정 파일에서 패키지별 제어 가능)
3.  **ERROR**: 시스템 장애, 데이터 유실 위험 등 즉시 확인이 필요한 이슈

---

## 5. 관리 및 활용 팁
*   **실시간 모니터링**: `tail -f logs/mega-cs-ai.log` 명령을 통해 AI 처리를 실시간 추적할 수 있습니다.
*   **파일 분리 이유**: `additivity="true"` 설정을 사용하여 개별 업무 로그(CS, Batch 등)와 전체 로그를 동시에 남기되, 문제 발생 시 특정 파일만 확인하여 가독성을 높였습니다.
