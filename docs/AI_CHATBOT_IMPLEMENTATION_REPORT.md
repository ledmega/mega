# AI 챗봇 위젯 구현 상세 보고서 (Feature Report)

이 문서는 LG전자 AI 챗봇 분석 결과를 바탕으로 **Mega CS AI 시스템**에 실제 구현된 플로팅 챗봇 위젯의 상세 사양과 기술 구조를 기록합니다.

## 1. 개요 (Overview)

*   **기능명**: AI 고객 지원 플로팅 위젯 (MEGA AI Chat)
*   **목적**: 외부 사용자(GUEST)가 로그인 없이도 AI와 실시간으로 소통하며 궁금한 점을 해결할 수 있는 채널 제공.
*   **구현 위치**: CS 대시보드([`cs/dashboard.html`](../webserver/src/main/resources/templates/cs/dashboard.html)) 좌측 상단.

---

## 2. 주요 구현 사항 (Key Features)

### 2.1 UI/UX 디자인
*   **Floating Action Button (FAB)**: 
    *   화면 **왼쪽 상단(Fixed)**에 위치하여 페이지 스크롤과 관계없이 상시 노출.
    *   보라색-핑크 그라데이션과 로봇 아이콘으로 시각적 강조.
*   **Chat Window**:
    *   클릭 시 스무스한 좌우 슬라이드 애니메이션과 함께 대화창 확장.
    *   모던한 카드 스타일의 헤더와 메시지 버블 디자인 적용.
*   **반응형 대응**: 모바일 및 태블릿 환경에서도 가독성을 해치지 않도록 적정 너비(360px) 유지.

### 2.2 지능형 인터랙션
*   **Quick Suggestions (Chips)**: 
    *   첫 진입 시 사용자가 자주 묻는 질문(배정, 설치, 장애 등)을 버튼 형태로 제공.
    *   클릭 한 번으로 즉시 질문이 전송되도록 구현.
*   **Visual Feedback**: 
    *   AI가 답변을 생성하는 동안 **Typing Indicator** (점 세 개가 튀는 애니메이션)를 노출하여 사용자가 응답 대기 중임을 인지하도록 유도.
*   **Auto-Scroll**: 메시지가 추가될 때마다 자동으로 하단으로 스크롤하여 최신 대화 유지.

### 2.3 기술적 메커니즘 (Technical Logic)
*   **API 연동**: 
    *   기존 시뮬레이션 엔드포인트인 `/api/cs/simulate/inbound`를 활용.
    *   `GUEST_USER` 식별자와 `PORTAL` 채널 코드를 사용하여 로그인 없이도 내부 AI 로직(RAG + Gemini) 가동.
*   **Reactive Security**: 
    *   `SecurityConfig` 내에서 `/api/cs/**` 경로에 대한 `permitAll()` 설정을 통해 비인증 사용자의 접근 권한 확보.
*   **Lightweight Implementation**: 
    *   추가적인 라이브러리 없이 **Vanilla JS**와 **Native CSS**만으로 구현하여 로딩 부하 최소화.

---

## 3. 테스트 및 작동 방법

1.  **브라우저 접속**: 프로젝트 실행 후 `http://localhost:8080/dashboard` 접속 (로그인 여부와 무관).
2.  **버튼 클릭**: 왼쪽 상단의 로봇 아이콘 클릭.
3.  **대화 시작**: 
    *   하단의 추천 칩을 클릭하거나 직접 질문 입력.
    *   예: "Agent는 어떻게 설치하나요?"
    *   예: "배정이 얼마나 걸릴까요?"
4.  **결과 확인**: AI가 우리 FAQ 데이터를 참고(RAG)하여 실시간으로 답변하는지 확인.

---

## 4. 향후 고도화 계획

*   **Rich Media 지원**: AI 답변 내에 이미지, 매뉴얼 PDF 링크 등을 포함할 수 있도록 렌더러 확장.
*   **세션 유지**: 브라우저 로컬 스토리지를 활용하여 페이지 새로고침 후에도 이전 대화 내역 유지.
*   **외부 서비스 버튼**: 챗봇 내에서 즉시 '담당자 연결'이나 'Redmine 일감 생성'이 가능한 액션 버튼 추가.
