## 1. 대시보드 차트 개별 시간 필터링 구현

**Q**: 현재 차트에 실시간 1h 12h 1d 옵션이 전체에 하나만 세팅 되어있는데 각 차트마다 따로따로 세팅 하는 게 맞는 것 같습니다.
**A**:
- **수정사항**: `DashboardApiController`, `MetricDataService`, `MetricDataRepository`를 수정하여 특정 `metricType` 또는 `metricName`으로 기간 한정 데이터를 가져오는 API를 구축했습니다.
- **UI**: `dashboard.html`에서 전역 필터를 제거하고 각 차트 상단 헤더에 개별 버튼을 배치했습니다.
- **버그 패치**: 개별 메트릭 명칭(CPU 사용률 등) 필터링 시 `metricType` 매칭 오류가 발생하던 부분을 해결하기 위해 API에 `metricName` 필터링 파라미터를 추가 반영했습니다.

## 2. OS 자원 수집 웹 설정 기능 제안 분석

**Q**: OS 자원 수집(메모리, CPU, 디스크 등) 설정을 웹 화면에서 동적으로 관리하고 싶습니다. (항목, 주기, 명령어, 옵션 등)
**A**:
- **결론**: 매우 타당하며 기술적으로 구현 가능합니다. 다만 보안 및 구조적 측면에서 몇 가지 제안을 드립니다.
- **보안**: 명령어를 웹에서 직접 입력받는 것은 보안 취약점(RCE) 위험이 있으므로, 에이전트 내부에서 정의된 `Enum` 항목을 선택하는 방식을 권장합니다.
- **구조**: 기존 `monitoring_config`는 서비스별 중심이므로, `os_monitoring_config` 테이블을 신규 설계하여 관리하는 것이 깔끔합니다.
- **반영**: 에이전트가 60초마다 설정을 Pull 해가는 기존 방식을 활용하면 구조 변경을 최소화할 수 있습니다.
- **특이사항**: "프로세스 이상 감지(지속 상승)" 기능은 단순 스냅샷만으로는 판단이 어려워, 에이전트 혹은 서버에서 시계열 데이터를 비교하는 추가 설계가 필요합니다.
